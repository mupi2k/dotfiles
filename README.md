Mike Porter's dotfiles
======================

Personal dotfiles managed with [chezmoi](https://chezmoi.io). Covers shell
configuration, aliases, git, tmux, neovim, karabiner, aerospace, and more.


Install
-------

```sh
brew install chezmoi
chezmoi init --source ~/.dotfiles git@github.com:mupi2k/dotfiles.git
chezmoi apply
```

If you already have the repo cloned at `~/.dotfiles`:

```sh
brew install chezmoi
chezmoi apply
```

chezmoi reads `~/.config/chezmoi/chezmoi.toml` to find the source directory.
If that file doesn't exist yet, create it:

```toml
sourceDir = "/Users/<you>/.dotfiles"
```


Structure
---------

| Path | Deploys to |
|------|-----------|
| `dot_*` | `~/.*` |
| `dot_config/` | `~/.config/` |
| `default-configs/` | sourced by `.bashrc` / `.zshrc` |
| `custom-configs/` | local overrides (gitignored) |

### Root dotfiles (`dot_*`)

| File | Purpose |
|------|---------|
| `dot_zshrc` | Zsh config — sources default-configs and custom-configs |
| `dot_bashrc` | Bash config — same sourcing mechanism as zshrc |
| `dot_gitconfig` | Git user config, aliases, and defaults |
| `dot_gitignore` | Global gitignore |
| `dot_aerospace.toml` | AeroSpace tiling WM — workspace/monitor layout, keybindings |

### `dot_zsh/`

Zsh support files deployed to `~/.zsh/`: shell function completions (`zfunc/`)
and the zsh-autosuggestions plugin.

### `dot_config/`

| Directory | Purpose |
|-----------|---------|
| `aerospace/` | AeroSpace scratchpad helper script |
| `bpytop/` | bpytop resource monitor config |
| `flake8` | Python linter config (max line length, complexity) |
| `gh/` | GitHub CLI config |
| `ghostty/` | Ghostty terminal — colors, theme |
| `git/` | Global gitignore (XDG path) |
| `htop/` | htop process monitor layout |
| `i3/` | i3 window manager config (Linux) |
| `karabiner/` | Karabiner-Elements keyboard remapping |
| `nvim/` | Neovim — lazy.nvim plugins, LSP, treesitter, Nord theme |
| `opencode/` | OpenCode AI assistant config |
| `tmux/` | tmux config and tmuxline status bar theme |

### `default-configs/`

Sourced automatically on shell startup unless overridden by a same-named file
in `custom-configs/`.

| File | Purpose |
|------|---------|
| `promptline.sh` | Shell prompt (generated by promptline.vim) |
| `functions.sh` | General-purpose shell functions |
| `git.sh` | Git aliases |
| `python.sh` | pyenv setup |
| `tmux.sh` | tmux aliases and color fix |
| `acc.sh` | AWS account lookup helpers |
| `ramsey-cli-aliases.sh` | Ramsey Solutions CLI aliases (loaded if `$USE_RAMSEY` set) |
| `pie.sh` | PIE environment helpers (loaded if `$USE_PIE` set) |
| `vader.sh` | Vader environment helpers (loaded if `$USE_VADER` set) |
| `screen.sh` | GNU screen aliases |
| `moxa.sh` | Moxa device helpers |
| `interactive.sh` | Interactive shell extras |
| `zsh-vim-mode.zsh` | Vim keybindings for zsh (zsh only) |


Custom Configs
--------------

Files in `default-configs/` are sourced automatically by both `.bashrc` and
`.zshrc`. To override one on a specific machine, drop a file with the same
name into `custom-configs/` — the default will be skipped in favour of yours.

Any `*.sh` or `*.zsh` files added directly to `custom-configs/` are also
sourced unconditionally, making it a good place for machine-local settings
and secrets (API keys, work-specific aliases, etc.). The `custom-configs/`
directory is gitignored.


Locale
------

Make sure `LC_ALL` is set:

```sh
export LC_ALL=en_US.UTF-8
```
